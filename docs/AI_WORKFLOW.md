# Mini RPG Dev — AI Agent 依頼ワークフロー

このプロジェクトで AI Agent に開発を依頼する際のルールです。

---

## 依頼テンプレート

各タスク（M0, M1, …）の依頼には、以下の 4 項目を**必ず**含めてください。

### 1. 現在の状態

すでに実装済みの機能を列挙します。

**例:**
```
現在の状態：
- M0 主人公移動
- M1 マップ表示
- M2 壁判定
- M4 カメラスクロール
- …
```

Agent は既存コードを読み、壊さないように差分を最小限にします。

---

### 2. 今回の仕様

今回追加・変更する内容を具体的に書きます。

- 何を実装するか
- ファイル構成の方針（新規ファイル / 既存ファイルの修正）
- 意図的に**今回やらないこと**（例: 会話はまだ不要）

---

### 3. 完了条件

「完了」とみなすためのチェックリストを書きます。

**例:**
```
完了条件：
- 主人公が壁に入れない
- 既存の移動が動く
- 変更したファイルと理由を説明する
```

---

### 4. 既存機能を壊さない

明示的に記載します。

**例:**
```
- 既存の移動、壁判定、カメラ、NPC 表示は壊さない
- コードはできるだけシンプルに、関数ごとに役割を分ける
```

---

## 依頼例

```markdown
M8（複数行会話）を実装してください。

現在の状態：
- M0〜M7 まで実装済み
- Space で会話開始 / 終了
- dialogue.js に NPC ごとのセリフ

今回の仕様：
- 1 回の会話で複数行のセリフを順番に表示
- Space で次の行へ進む
- 最後の行の次の Space で閉じる

完了条件：
- npc1 の会話が 2 行以上表示できる
- 会話中は移動しない
- 既存機能は正常に動く
- 最後に変更したファイルと理由を説明する
```

## 作業後のフロー

Agent による実装完了後、以下の流れで確認・記録します。

### 1. 動作確認

- [ ] ブラウザで `index.html` を開く（または `python3 -m http.server` で配信）
- [ ] 今回の完了条件をすべて満たすか確認
- [ ] 既存機能（移動・壁判定・カメラ・NPC・会話）が壊れていないか確認
- [ ] 更新後は **Cmd + Shift + R**（スーパーリロード）でキャッシュをクリア

### 2. 開発ログの更新

- [ ] `DEVLOG.md` に作業内容・問題と修正を追記
- [ ] `ROADMAP.md` の該当マイルストーンを ✅ に更新

### 3. Git コミット

```bash
git status
git diff
git add .
git commit -m "feat: M8 複数行会話を実装"
```

**コミットメッセージの目安:**
- `feat:` 新機能
- `fix:` バグ修正
- `docs:` ドキュメントのみ
- `style:` 見た目調整（ロジック変更なし）

### 4. Git プッシュ

```bash
git push origin main
```

## Asset Workflow

AI だけでスプライト制作を行う際の、Asset 共通フローです。
今後すべての Asset を同じ手順で制作します。

```
Concept Art
    ↓
Review
    ↓
Approved
    ↓
32×32 Sprite
    ↓
Review
    ↓
Approved
    ↓
4 Direction
    ↓
Walk Animation
    ↓
Game Integration
```

### 各ステップの概要

| ステップ | 内容 |
|----------|------|
| Concept Art | コンセプトアートを制作する |
| Review | 世界観・キャラ設定との整合を確認する |
| Approved | 採用を決定し、`CHARACTER_BIBLE.md` 等のデザイン履歴に記録する |
| 32×32 Sprite | 採用デザインを基準にドット絵（32×32）を制作する |
| Review | `ART_BIBLE.md` のドットルール・色設計に沿っているか確認する |
| Approved | スプライト採用を決定する |
| 4 Direction | 上下左右の 4 方向スプライトを制作する |
| Walk Animation | 歩行 4 コマ・待機 1 コマのアニメーションを制作する |
| Game Integration | `sprites/` に配置し、ゲームへ組み込む |

### 参照ドキュメント

- [ART_BIBLE.md](./ART_BIBLE.md) — 世界観・色・ドットルール
- [CHARACTER_BIBLE.md](./CHARACTER_BIBLE.md) — 主人公設定・スプライト仕様・デザイン履歴
- [AI_DASHBOARD.md](./AI_DASHBOARD.md) — 現在の Asset 作業状況

## プロジェクトの設計方針

Agent に守ってほしい原則です。

| 原則 | 内容 |
|------|------|
| 最小差分 | 依頼された範囲だけ変更する |
| 役割分担 | 入力 `input.js`、マップ `map.js`、NPC `npc.js` などファイルごとに責務を分ける |
| 拡張性 | データ（マップ・NPC・会話）は配列やオブジェクトに追加するだけで増やせる構造にする |
| 座標 | 移動・当たり判定はワールド座標、描画はカメラ変換を通す |
| キャッシュ | script / CSS 更新時は `index.html` の `?v=` を更新する |
