# Hero Readability Study — 主人公が「主人公」だと分かる理由

Asset-09

## 目的

名作 RPG の主人公が **一瞬で主人公だと認識される理由** を、ドットの描き方ではなく **デザインの考え方** として分析する。

[REFERENCE_BOARD.md](./REFERENCE_BOARD.md) が「技法・ルール」であるのに対し、本ドキュメントは **なぜそう設計するか** を扱う。

**関連ドキュメント**

- [REFERENCE_BOARD.md](./REFERENCE_BOARD.md) — 技法分析・Mini RPG Dot Design Rules
- [ART_BIBLE.md](./ART_BIBLE.md) — 世界観・配色
- [CHARACTER_BIBLE.md](./CHARACTER_BIBLE.md) — 主人公設定

## 分析の前提

フィールドスプライトは **数ミリ秒〜1秒未満** で認識される。  
プレイヤーは細部を「見る」のではなく、 **色の塊・形の塊・動き** で判断する。

主人公 Readability の核心は次の 3 点に集約される。

1. **最初の 0.5 秒で何が目に入るか**（第一印象）
2. **情報を捨てて何を残すか**（優先順位）
3. **動いても形が壊れないか**（歩行前提）

---

## 1. 第一印象 — 人は最初にどこを見るか

### 共通パターン

| 順位 | 視線の着地点 | 理由 |
|------|-------------|------|
| 1 | **髪・頭部の色面** | 全高の 35〜50% を占め、背景（緑・茶）と最も色相が離れやすい |
| 2 | **上半身の大色面** | 服・鎧の色が 2 番目に大きい面積を持つ |
| 3 | **非対称要素** | 武器・マント・バッグ・帽子が「人間形」の記憶を固定する |
| 4 | **顔** | 小さいが、目・前髪のコントラストで「誰」感を与える |

顔が **最初** ではない。  
「赤い塊が歩いている」→「人だ」→「主人公だ」の順で認識される。

### 作品別の第一印象

| 作品 | 最初に目に入るもの | 設計意図 |
|------|-------------------|----------|
| **クロノ・トリガー** | 赤髪の塊 | 髪色だけで Crono を特定。顔より先に髪 |
| **DQIII** | 頭部全体（帽・盔） | 髪を捨て、頭部シルエット 1 つに集約 |
| **DQVI** | 髪色 + 上着の色面 | III より色面が増え、個性が出る |
| **FFVI** | 髪型 + マント/コートの外縁 | パーティ内で **形** で差別化 |
| **聖剣伝説3** | 髪 + 肩〜腰の装備塊 | クラス（剣/槍/魔法）が **シルエット** で伝わる |

### デザインの考え方

第一印象は **「詳細」ではなく「最大コントラストの塊」** で決まる。  
Mini RPG でも、最初に目に入るのは **赤髪** であるべきで、目や口の細部ではない。

---

## 2. シルエット — 真っ黒な影だけでも分かる理由

### 共通パターン

シルエットだけで判別できるのは、次の **外縁の形** が固定されているから。

- **頭部：** 髪の spike・帽・ポニテなど、他キャラと **異なるトップ形状**
- **肩：** 幅・マント・肩当てで **逆三角形 or 台形** が作られる
- **非対称：** 武器・バッグ・マントが **片側だけ** 膨らむ
- **足元：** 足の開き方・マントの裾で **接地感** が出る

### 作品別

| 作品 | 影だけで分かる理由 |
|------|-------------------|
| **クロノ・トリガー** | 髪 spike + 細身 + 片側に剣/髪のはみ出し |
| **DQIII** | 極小だが **帽の丸み + 縦長** が全員共通フォーマットの中で差 |
| **DQVI** | 前髪 1 列 + 肩幅で **III より個別性** |
| **FFVI** | マント・大きな髪・コート裾 — **外縁の複雑さ** が ID |
| **聖剣伝説3** | 武器サイズ・マント・髪の長さで **クラス別影** |

### デザインの考え方

「中身を描いてから輪郭を付ける」のではなく、 **影の形を先に決める** 。  
Mini RPG 主人公の影テスト：髪トップ・肩ライン・右腰バッグの 3 点が **他 NPC と重ならない** こと。

---

## 3. 視線誘導 — 視線はどの順番で移動するか

### 典型フロー（フィールド・正面）

```
髪（色） → 顔（目） → 胸（服の色） → 小物（バッグ/武器） → 足（接地）
```

### 作品別

| 作品 | 視線の流れ | 意図 |
|------|-----------|------|
| **クロノ・トリガー** | 赤髪 → 目 → 青服 → 剣/髪先 | 顔で止め、服で「味方」確認 |
| **DQIII** | 頭（帽）→ 体（単色）→ 足 | 顔を飛ばして **記号** として読ませる |
| **DQVI** | 髪 → 目 → 服 → 足 | III と CT の中間。目で **人間性** |
| **FFVI** | 髪/帽 → マント外縁 → 顔 → 足 | マントが **視線を体に引き込む** |
| **聖剣伝説3** | 髪 → 肩装備 → 武器 → 足 | 装備で **役割** を読ませる |

### デザインの考え方

視線誘導は **意図的な「止まりポイント」** の設計。  
Mini RPG では **緑スカーフ（首）** が髪→服の間に **1 コマ止め** となり、旅人感と視線の段差を作る。

---

## 4. 情報の優先順位 — 何を強調しているか

情報量ではなく **捨てた情報** が Readability を決める。

### 作品別 · 優先順位

**クロノ・トリガー**

1. 赤髪
2. 顔（目）
3. 青服
4. （フィールドでは剣は省略されがち）

**DQIII**

1. 頭部形状（帽）
2. 体の単色
3. 足
4. 顔・武器・影 — **省略**

**DQVI**

1. 髪色
2. 服の色面
3. 顔（目 2px）
4. 小物 — **最小**

**FFVI**

1. シルエット（髪 + マント）
2. 髪色
3. 服色
4. 顔 — **相対的に小さく**

**聖剣伝説3**

1. 髪 + 装備シルエット
2. 武器の有無・サイズ
3. 服色
4. 顔の細部

### デザインの考え方

**強調 = 面積 × 彩度 ×  uniqueness** の積。  
Mini RPG 主人公の優先順位（確定）：

```
赤髪 → 顔（目）→ 青服 → 緑スカーフ → バッグ → 足
```

## 5. 主人公らしさ — NPC との差

### 主人公だけが持つ特徴（5 作品共通）

| 特徴 | 内容 |
|------|------|
| **彩度** | 背景・NPC より 1 段高い |
| **色数** | やや多いが、 **使う色は少ない（3〜4 系統）** |
| **非対称小物** | バッグ・武器・マントで「旅・戦」の物語を暗示 |
| **頭部比率** | やや大きめ（ chibi ）で **視線を引く** |
| **動き** | 待機・歩行に **余白** がある（完全静止でない） |

### 作品別 · NPC との差

| 作品 | 主人公の差 |
|------|-----------|
| **クロノ・トリガー** | パーティ全員が個性強。主人公は **最も中性的で短髪赤** |
| **DQIII** | 全員同フォーマット。**色** だけが差（実質プレイヤー=全員） |
| **DQVI** | NPC より **髪・服の色面が大きい** |
| **FFVI** | パーティ内 **最も明るい色** を主役に |
| **聖剣伝説3** | 6 人選択制。**クラス＋髪** で即判別 |

### デザインの考え方

Mini RPG は **単一主人公** のため、DQIII 型（色だけの差）ではなく **CT / FFVI 型（シルエット + 彩度）** を採用。  
NPC は **色数・彩度・非対称小物を 1 段下げる**（[REFERENCE_BOARD.md](./REFERENCE_BOARD.md) ルール 10・11）。

## 6. 色の役割 — 色数ではなく使い方

### 3 層モデル

| 層 | 役割 | 例 |
|----|------|-----|
| **主役色（1〜2）** | 即認識。画面の 15〜25% | クロノ=赤、DQ=服色、Mini RPG=赤+青 |
| **補助色（1〜2）** | 体積・部位の区切り | 肌、影 1 段、パンツ |
| **アクセント色（1）** | 視線止め・性格暗示 | 緑スカーフ、金バックル、帽 |

### 作品別

| 作品 | 主役色 | 補助色 | アクセント |
|------|--------|--------|-----------|
| **クロノ・トリガー** | 赤髪 + 青服 | 肌・影 | ベルト・肌の明部 |
| **DQIII** | 服 1 色 | 肌（最小） | 帽の縁 |
| **DQVI** | 髪 + 服 | 肌・影 | 前髪 |
| **FFVI** | 髪 + マント/服 | 肌 | 装飾 1 色 |
| **聖剣伝説3** | 髪 + 装備 | 服・肌 | 武器・リボン |

### デザインの考え方

**主役色は 2 つまで。** 3 つ目はアクセントに回す。  
Mini RPG：主役 = **赤 + 青**、補助 = 肌・茶（パンツ・バッグ・靴）、アクセント = **緑**（CHARACTER_BIBLE 準拠）。

## 7. 32×32 で重要な要素 — 省略と残す

### 5 作品が省略するもの

- 鼻・口（または 1px まで）
- 指・関節
- 服の模様・ボタン（複数）
- 武器の詳細（フィールド）
- 影 2 段以上

### 5 作品が絶対残すもの

- **髪色の面積**
- **服の色面（胸〜腰）**
- **頭と体の接続（首・肩）**
- **足 2 列（または交互の暗示）**
- **輪郭 1px**

### Mini RPG（32×32）の判断

| 残す | 省略 |
|------|------|
| 赤髪 spike | 髪の陰影 2 段 |
| 目 2×2 px | 鼻 |
| 緑スカーフ 1 列 | 口（または 1px のみ） |
| 青ベスト + 白シャツ中心 | ボタン・模様 |
| バッグ（片側） | bedroll・細装飾 |
| 足 2 列 + 靴 | 指・ソール詳細 |

32×32 は DQIII の 2 倍のキャンバス。**III の省略度は使わない。** CT / DQVI の **顔 + 色面** を採用する。

## 8. 歩行を前提にした設計 — 歩いても崩れない理由

### 共通原則

1. **重心が中央** — 左右非対称は **小物だけ**
2. **足は 2 列** — 交互に動かす余白
3. **腕は体から分離** — スイング用 1〜2px
4. **髪・マントは従属** — 体の 1px 遅れ（主シルエットを変形しない）
5. **頭部は体と一体** — 首を細くしすぎない

### 作品別

| 作品 | 歩行安定の設計 |
|------|-------------|
| **クロノ・トリガー** | 細身・髪 spike 固定・体ボブ 1px |
| **DQIII** | 足開閉のみ・上下動最小 |
| **DQVI** | 足 + 体 1px ボブ |
| **FFVI** | マントは独立アニメ・体は単純 |
| **聖剣伝説3** | 多コマだが **芯の体は不変** |

### デザインの考え方

歩行で崩れるのは **「アニメ用に描いた細部」** 。  
Mini RPG Master は **静止画で歩行の芯（足の gap・腕の位置）を確保** し、髪・バッグだけ後から揺らす（Asset-05.3 検証済み）。

## 9. Mini RPG への反映 — 追加ルール

[REFERENCE_BOARD.md](./REFERENCE_BOARD.md) の Dot Design Rules **に加え**、Readability 観点のルールを定義する。

| # | ルール | 理由 | 参考 |
|---|--------|------|------|
| R1 | **第一印象は赤髪。** 1x 表示で最も目立つ面積を髪が占める | 5 作品共通：頭部色面が最初 | CT, FFVI |
| R2 | **影シルエットテスト必須。** 単色塗りでも主人公と分かること | 外縁 ID | 全作品 |
| R3 | **視線止めは首の緑。** 髪→スカーフ→服の 3 段 | 情報優先順位 | Mini RPG 独自 |
| R4 | **強調は 3 要素まで：** 赤髪 / 青服 / バッグ | 情報過多を防ぐ | CT, DQVI |
| R5 | **NPC より彩度 +1、色数 −2〜4** | 主人公らしさ | FFVI |
| R6 | **主役色は赤+青のみ。** 緑はアクセント 1 列 | 色の役割 3 層 | ART_BIBLE |
| R7 | **顔は 2 番目に読む。** 目 2×2 を必須、鼻口は省略可 | 視線誘導 | CT, 聖剣3 |
| R8 | **非対称はバッグ 1 つのみ。** 武器は Walk 以降 | 歩行安定 | CT |
| R9 | **1x 5 秒テスト：** 草/木/白背景で「誰？」と自問 | 実環境 Readability | 実務 |
| R10 | **Master 更新時は Readability 再チェック** | 細部追加が認識を下げる | Asset-08 教訓 |

### Hero Readability Checklist（制作前後）

- [ ] 1x で **赤→顔→青→緑→バッグ** の順に読めるか
- [ ] 単色シルエットで **髪 spike + 肩 + バッグ** が分かるか
- [ ] 背景 3 種（草・白・黒）で **埋もれない** か
- [ ] 足 2 列が **歩行 gap** として空いているか
- [ ] NPC 試作と並べ **主役色が 1 段目立つ** か

## 作品別 · Readability サマリー

| 作品 | 一瞬で分かる理由（1 文） |
|------|-------------------------|
| **クロノ・トリガー** | 赤髪と青服の **2 色記号** が強すぎて、顔が小さくても主人公だと分かる |
| **DQIII** | 情報を捨て **帽+単色体** だけにした結果、最小ドットで「人」になる |
| **DQVI** | III に **目と色面** を足し、記号から人物へ |
| **FFVI** | **マント+髪** の外縁で群衆の中から主役を拾える |
| **聖剣伝説3** | **装備シルエット** がクラスを語り、動いても芯が残る |

## Mini RPG 主人公 · 確定 Readability モデル

Asset-08 `player_master_32_redesign_v1.png` は以下のモデルに沿う。

```
【第一印象】  赤髪 spike
【視線】      髪 → 目 → 緑スカーフ → 青ベスト → バッグ → 足
【シルエット】 髪 top / 肩ライン / 右腰バッグ
【色】        主=赤+青 / 補=肌・茶 / アク=緑
【歩行芯】    足 2 列 / 腕分離 / バッグ従属
```

Idle / Walk / 4 方向制作時は、**Readability を落とす変更（細部追加・彩度低下・対称化）を禁止** する。
