## 分析の前提

### Mini RPG のキャンバス

| 項目 | Mini RPG | 参考作品（概ね） |
|------|----------|------------------|
| 主人公キャンバス | **32×32** | 16×16〜16×24 が多い |
| タイル | 16×16 | 16×16 |
| 視点 | 真上寄り 3/4 | 真上寄り 3/4 |

Mini RPG は SFC 時代のフィールドスプライトより **1段大きい** 。  
小ささでの読みやすさの工夫は引き継ぎつつ、顔・小物・待機アニメに余裕を持たせる設計とする。

### 分析の観点（共通）

各作品について以下を整理する。

- 頭身 / 頭部サイズ / 肩幅 / 腕・脚の比率
- 顔パーツの配置
- 髪・配色・色数・輪郭線・影
- シルエット
- 歩行アニメを前提とした設計
- 小物（バッグ・マント等）

## 作品別分析

### クロノ・トリガー（Chrono Trigger）

**参照フォルダ:** `assets/reference/chrono/`

#### 概要

フィールド用キャラクターは、タイル 16×16 グリッド上で **縦にやや長い chibi** として配置される。  
歩行・待機の **アニメーション品質** が高く、髪・マント・武器の **従属モーション** が特徴的。

#### 分析

| 項目 | 傾向 |
|------|------|
| 頭身 | 約 **2.5〜3 頭身**（フィールド） |
| 頭部サイズ | 全高の **40〜45%** 。頭が大きく、人物認識は髪色と頭部シルエットで行う |
| 肩幅 | 頭幅の **0.8〜1.0 倍** 。コンパクト |
| 腕・脚 | 腕は短く、振りは **1〜2px** の単純なスイング。脚は交互に **1px 前後** |
| 顔パーツ | 目は **1〜2px** 、口は省略または 1px。正面は目の水平配置が基本 |
| 髪 | 髪色でキャラ識別。**外側に 1px はみ出し** でボリュームを出す |
| 配色 | キャラごとに **3〜5 色のランプ**（ベース・影・ハイライト） |
| 使用色数 | 1 キャラ **12〜20 色程度**（アウトライン含む） |
| 輪郭線 | **黒に近い暗色** 。髪と服の境界は明確 |
| 影色 | ベースより **1 段暗い色 1 つ** が基本。過度なグラデーションなし |
| シルエット | 髪のトップと肩ラインが最重要。横向きでも **鼻・顎を出さず** 頭部塊で表現 |
| 歩行アニメ | **3〜4 コマ** 。体全体の **上下 1px ボブ** + 足の交互 + 腕振り |
| 小物 | マント・髪は **1 コマ遅れ** で追随。主シルエットを壊さない範囲 |

#### 技法メモ

- 「動きの主役」と「追随パーツ」を分離する思想が強い
- 待機時も **完全静止にしない**（呼吸・揺れの余地）
- 小さくても **髪色＋上半身の色面** で判別可能

### ドラゴンクエストIII（SFC）

**参照フォルダ:** `assets/reference/dq3/`

#### 概要

フィールドスプライトは **極小・極限単純化** が特徴。  
「最小ドット数で人間に見せる」設計の典型。

#### 分析

| 項目 | 傾向 |
|------|------|
| 頭身 | 約 **2 頭身** に近い |
| 頭部サイズ | 全高の **50% 前後** 。頭が支配的 |
| 肩幅 | 狭い。**頭より肩がはみ出さない** ことも多い |
| 腕・脚 | **各 1 列** で表現。関節表現はほぼ省略 |
| 顔パーツ | 正面は **目 2 点** のみ。口・鼻なしが基本 |
| 髪 | **帽・盔** などでシルエットをまとめる。髪単体より「頭部の形」 |
| 配色 | **平坦色** 。影は最小限 |
| 使用色数 | **8〜12 色** 程度に抑える傾向 |
| 輪郭線 | 黒。内外の色面境界がはっきり |
| 影色 | 1 段暗色のみ、または省略 |
| シルエット | **正面＝縦長の楕円＋足** 。横向きは幅 1px の増減で向き表現 |
| 歩行アニメ | **2〜4 コマ** 。足の開閉が主。上下動は小さい |
| 小物 | 盾・武器は **1〜2px** の記号として付加 |

#### 技法メモ

- **色数削減** と **形の単純化** を優先
- 32×32 でも「DQ 的に読ませる」なら、パーツ数より **色面ブロック** を優先
- Mini RPG はキャンバスが大きいため、DQ3 同等の省略は **顔・小物が消える** — 部分的参考にとどめる

---

### ドラゴンクエストVI（SFC）

**参照フォルダ:** `assets/reference/dq6/`

#### 概要

DQIII 系統を継承しつつ、**頭身・色使い・ディテール** が一段階上がる。  
「小さいが、人物性が出る」バランスの参考になる。

#### 分析

| 項目 | 傾向 |
|------|------|
| 頭身 | 約 **2.5〜3 頭身** |
| 頭部サイズ | 全高の **40〜45%** |
| 肩幅 | DQIII より **やや広い** 。上着の色面が見える |
| 腕・脚 | 腕の付け根と足首に **1px の段差** |
| 顔パーツ | 目 **2px 幅** のことが増える。横向きは **1 目** |
| 髪 | 髪型でキャラ差別。**前髪 1 列** が表情に効く |
| 配色 | 服・髪に **2 段階の明暗** |
| 使用色数 | **12〜18 色** |
| 輪郭線 | 黒〜暗色 |
| 影色 | ベースの **約 15〜25% 暗** |
| シルエット | 頭部＋肩の **逆三角形** が安定 |
| 歩行アニメ | **4 コマ** が基本に近い。体の上下動あり |
| 小物 | マント・リボン等、**1px 幅** の付属物 |

#### 技法メモ

- DQIII と CT の **中間的読みやすさ**
- 主人公の **服の色面** を大きく取ると NPC との判別が容易
- Mini RPG の 4 頭身は DQVI より高いが、**「目 2px・前髪 1 列」** の考え方は流用できる

### ファイナルファンタジーVI（FFVI）

**参照フォルダ:** `assets/reference/ff6/`

#### 概要

フィールドは比較的小さなスプライトだが、**個性（髪・衣装・シルエット）** の差別化が強い。  
「群衆の中で主役を見つける」ための **コントラスト設計** が参考になる。

#### 分析

| 項目 | 傾向 |
|------|------|
| 頭身 | 約 **3 頭身** |
| 頭部サイズ | 全高の **35〜40%** |
| 肩幅 | キャラにより差。**マント・コート** で幅を出す |
| 腕・脚 | 歩行時 **腕 1px スイング** 。脚は交互 |
| 顔パーツ | 正面は **目＋髪の影** で顔を作る。口は非表示が多い |
| 髪 | **髪色・髪型** が最大の識別子。大きめの塊 |
| 配色 | パーティ内で **色相を分散** 。主役級は明るい色 |
| 使用色数 | **15〜22 色** |
| 輪郭線 | 黒。髪と背景の分離に強く依存 |
| 影色 | 1 段＋ **髪用ハイライト 1 色** |
| シルエット | **頭部と肩ライン** で個性。縦長 vs 横長の対比 |
| 歩行アニメ | **4 コマ** 。マントは **独立した揺れ** |
| 小物 | マント・帽子・武器。**シルエット外縁** を変える |

#### 技法メモ

- 主人公は **彩度・明度で背景より一段目立つ** — Mini RPG ART_BIBLE と一致
- マント・コートは **主シルエットの延長** として扱う（バッグにも応用可）
- 「黒アウトライン」は FF 系だが、Mini RPG は **濃茶** に置換（世界観に合わせる）

### 聖剣伝説3（Seiken Densetsu 3）

**参照フォルダ:** `assets/reference/seiken3/`

#### 概要

フィールドアニメーションの **滑らかさ・従属モーション** が際立つ。  
歩行・待機における **髪・マント・装飾の追随** は Asset-05.3 以降の参考として重要。

#### 分析

| 項目 | 傾向 |
|------|------|
| 頭身 | 約 **3〜3.5 頭身** |
| 頭部サイズ | 全高の **35% 前後** |
| 肩幅 | やや広め。**上着の色面** がはっきり |
| 腕・脚 | 歩行で **膝の段差 1px** 。腕振りは 2px 程度 |
| 顔パーツ | 正面は **目 2〜3px** 。横向きは **鼻 1px** 程度 |
| 髪 | **束・房** として描き、動きで **1px 遅延** |
| 配色 | **暖冷のバランス** 。影に **紫〜茶** を混ぜることがある |
| 使用色数 | **18〜24 色** |
| 輪郭線 | 暗色。髪先は **アウトライン省略** も |
| 影色 | **2 段** まで（Mini RPG は 1 段に抑える） |
| シルエット | 髪・マントで **外縁を複雑化** しつつ、芯は単純 |
| 歩行アニメ | **4〜8 コマ** 。待機も **複数コマ** |
| 小物 | マント・リボン・武器。**慣性 1 コマ遅れ** |

#### 技法メモ

- **主動作 → 従属パーツ** の順で設計（Asset-05.3 down 待機と同思想）
- 待機は **1px 呼吸＋髪/小物 1px 揺れ** が有効
- 色数は多いが、Mini RPG は **朝の海辺** 世界観のため影の彩度は抑える

---

## 共通点（5 作品横断）

1. **頭部優先** — 全高の 35〜50% を頭が占める。人物認識は頭部シルエットと髪色が担う。
2. **顔は省略** — 32px 以下では鼻・口を省略または 1px。目は **1〜3px** で十分。
3. **影は 1 段** — ベース色より 1 段暗い色が基本。多段グラデーションは限定的。
4. **輪郭線で分離** — 背景・タイルとの分離に暗色アウトライン（Mini RPG は茶色）。
5. **4 方向設計** — 上下左右で **別絵** 。単純な回転ミラーは横向きのみ許容。
6. **歩行は 4 コマ前後** — 足の交互＋体の上下 **1px ボブ** が定番。
7. **待機も静止画 1 枚ではない** — 呼吸・揺れで「生きている」感じ。
8. **小物は従属** — バッグ・マントは主シルエットを壊さず **1px 遅れ** で動かす。
9. **主人公は目立つ** — 明度・彩度・シルエットサイズで背景より一段上。
10. **シルエットファースト** — 細部より **外縁の読みやすさ** を優先。

## 相違点

| 観点 | 最小（DQIII） | 中間（DQVI / CT） | リッチ（FFVI / 聖剣3） |
|------|---------------|-------------------|------------------------|
| 頭身 | 〜2 頭身 | 2.5〜3 頭身 | 3〜3.5 頭身 |
| 色数 | 8〜12 | 12〜18 | 18〜24 |
| 顔の省略度 | 最大 | 中 | やや詳細 |
| 待機アニメ | 少ない / 1 コマ | 2〜4 コマ | 4 コマ以上 |
| 小物の動き | ほぼなし | 1px 揺れ | 慣性・遅延あり |
| アウトライン | 黒固定 | 黒〜暗色 | 暗色＋省略箇所 |

Mini RPG は **中間〜ややリッチ** を採用する。

- キャンバス 32×32 → DQIII ほどの省略はしない
- 世界観（朝・海辺）→ FFVI / 聖剣3 ほどの高彩度影は使わない
- 主人公 4 頭身 → CT / 聖剣3 寄りだが、**chibi 可愛さ** は CHARACTER_BIBLE に合わせる

## Mini RPG に採用するルール

既存 [ART_BIBLE.md](./ART_BIBLE.md) / [CHARACTER_BIBLE.md](./CHARACTER_BIBLE.md) / Master Sprite と **矛盾しない** 範囲で、参考作品から採用する方針。

| # | ルール | 採用理由 | 主な参考 |
|---|--------|----------|----------|
| 1 | キャンバス **32×32**、有効描画域 **24〜28px 高** | 顔・小物・待機アニメに余裕。タイル 2 枚分以内 | CT, 聖剣3 |
| 2 | 頭身 **約 4 頭身** | 既存 Master Sprite / CHARACTER_BIBLE と一致 | Mini RPG 独自 |
| 3 | 頭部 **全高 35〜40%** | 人物認識と chibi バランス | CT, FFVI |
| 4 | 目 **2px 幅**（正面）、横向き **1 目 1〜2px** | 小サイズでも視認性 | DQVI, 聖剣3 |
| 5 | 鼻・口は **省略または 1px** | ドット不足時のノイズ防止 | DQIII, CT |
| 6 | アウトライン **濃い茶 (92,64,51)** 、黒不使用 | 朝の海辺・暖色世界観 | ART_BIBLE |
| 7 | 影 **1 段暗色のみ** | タイル・背景との調和 | DQIII, CT |
| 8 | 主人公色数 **15〜20 色**、NPC は **12〜16 色** | 主役を目立たせつつ管理可能 | FFVI |
| 9 | 主人公のみ **彩度 +1 段** | 背景に埋もれない | FFVI, ART_BIBLE |
| 10 | 歩行 **4 コマ** ＋ **上下 1px ボブ** | SFC 標準。実装コストと見た目のバランス | 全作品 |
| 11 | 待機 **4 コマ** ＋ **1px 呼吸** | Asset-05.3 で検証済み | CT, 聖剣3 |
| 12 | 髪・バッグは **主動作 1 コマ遅れ** で 1px 揺れ | 従属モーション。シルエット維持 | CT, 聖剣3 |
| 13 | 4 方向は **個別描き**（up は mirror 不可） | バッグ・前髪の向きが破綻しない | 全作品 |
| 14 | 横向き right は left **反転＋バッグ位置修正** | 制作効率。小物だけ手直し | 実務慣行 |
| 15 | シルエット **髪トップ＋肩ライン＋バッグ** で固定 | 主人公識別要素 | CHARACTER_BIBLE |
| 16 | 小物（バッグ）は **主シルエット外 1〜2px** まで | 特徴になるが主張しすぎない | Asset-04 レビュー |
| 17 | アニメ **0.12 秒/コマ**（待機） | Asset-05.3 プレビューで OK | Mini RPG 検証値 |
| 18 | Master Sprite 最新版を **最優先参照** | 文章ルールよりビジュアル基準 | ART_BIBLE |

## Mini RPG Dot Design Rules

AI・制作者がスプライト制作時に参照する **確定ルール集**（10〜20 項目）。

### キャンバス・比例

1. **32×32 RGBA PNG、背景透明。**
2. **有効描画域は縦 24〜28px。** 上下 2〜4px は余白として確保する。
3. **頭身は約 4 頭身。** 頭部は全高の 35〜40% を目安とする。
4. **肩幅は頭幅の 0.9〜1.1 倍。** 上着の色面で肩ラインをはっきりさせる。

### 顔・髪

5. **正面の目は 2px 幅。** 横向きは 1 目 1〜2px。鼻・口は省略または 1px まで。
6. **髪色・髪型はキャラ識別の第一要素。** 主人公は鮮やかな赤（Master Sprite 準拠）。
7. **髪の外縁 1px はみ出し** でボリュームを出すが、シルエット幅の増加は **左右 1px 以内**。

### 色・線

8. **アウトラインは濃い茶色。** 純黒 `#000000` は使わない。
9. **各パーツの影はベースより 1 段暗い色 1 つのみ。** 2 段以上のグラデーションは使わない。
10. **主人公の配色数は 15〜20 色（アウトライン含む）。** NPC は 12〜16 色を目安。
11. **主人公のみ世界より彩度を 1 段高くする。** 背景タイルと同系色の服は避ける。

### シルエット

12. **シルエットファースト。** 細部の前に、髪トップ・肩・足元の外縁が 1x 表示で読めることを確認する。
13. **小物（バッグ等）はシルエットの一部として設計** するが、主張は **主人公の 20% 以下** の面積。

### アニメーション

14. **歩行は 4 コマ。** 足の交互＋体全体 **1px 上下ボブ** を基本とする。
15. **待機は 4 コマ。** 全体 **1px 呼吸**（下→上→静止）を基本とする。
16. **髪・バッグ・マントは主動作の 1 コマ遅れで 1px 揺れる。** 主シルエットを 2px 以上変形しない。
17. **待機 1 コマあたり 0.12 秒** を初期値とする。速さ調整は 0.10〜0.16 秒の範囲。

### 方向・制作フロー

18. **上下左右は個別に描く。** up（背面）は left の反転では作らない。
19. **right は left を反転し、バッグ・前髪・非対称小物のみ手直し** する。
20. **新規 Asset は必ず Master Sprite（`player_master_v1.png` 最新版）と並べて確認** する。文章ルールと Master Sprite が矛盾する場合、Master Sprite を優先する。

## 今後の参照方法

1. スプライト制作前に本ドキュメント **Mini RPG Dot Design Rules** を読む。
2. [HERO_READABILITY.md](./HERO_READABILITY.md) の Checklist で **主人公らしさ** を確認する。
3. Master Sprite を開き、色・頭身・シルエットを目視確認する。
4. 必要に応じて `assets/reference/<作品>/` にローカル参考資料を追加する（コミット不要）。
5. 完成後は [CHARACTER_BIBLE.md](./CHARACTER_BIBLE.md) のデザイン履歴に記録する。
