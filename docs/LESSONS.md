## 2026-06-26

### Git

- 小さなマイルストーンごとに commit する
- 動作確認後に push する
- コミットメッセージは `feat:` / `fix:` / `docs:` など prefix で意図を伝える
- 1 コミット = 1 マイルストーン の粒度がレビューしやすい

### Cursor Agent

- **「現在の状態」** を書く — 実装済み機能を列挙し、Agent が既存コードを把握しやすくする
- **「今回の仕様」** を書く — 追加・変更内容と、今回やらないことを明確にする
- **「完了条件」** を書く — チェックリスト形式で「完了」の定義を揃える
- **「既存機能を壊さない」** を書く — 移動・壁判定・カメラなど維持すべき機能を明示する
- 依頼の 4 項目がそろっていると、差分が小さく、意図どおりの実装になりやすい

### JavaScript

- モジュールを分けると拡張しやすい
- 機能ごとにファイルを分ける（例: `input.js`, `map.js`, `npc.js`, `dialogue.js`）
- ゲームループ（`update` → `render`）と各機能の責務を分離する
- ワールド座標（移動・当たり判定）と画面座標（カメラ変換後の描画）を分ける
- データ（マップ・NPC・会話）を配列・オブジェクトに置き、ロジックと分離する

### バグ対応

- **キャッシュ問題**が起こることがある
  - ブラウザが古い `input.js` を読み込み、新しい API（`consumeAction`）が未定義になる
  - 対策: script タグに `?v=...` を付ける、スーパーリロード（`Cmd + Shift + R`）する
- **真っ暗な画面**は JS 例外が原因だった
  - `update()` 内で例外 → `render()` まで到達しない → キャンバスが描画されない
  - 対策: 開発者ツールの Console を確認、`gameLoop` に try-catch を入れる、防御的チェックを書く
- **機能追加時に既存関数を消してしまう**ことがある
  - M6 実装時に `interaction.js` から `getNpcAtFront` が削除され、Space キーが効かなくなった
  - 対策: ファイルを書き換える際は、既存の公開関数一覧を確認する
- **localhost が起動していない**ことがある
  - `python3 -m http.server` を実行してからブラウザで確認する
  - ポートが使用中の場合は別ポートを使う、または既存プロセスを確認する

### 設計

- マイルストーン（M0, M1, …）で段階的に機能を足すと、問題の切り分けがしやすい
- 当たり判定・描画・入力・会話を別ファイルにすると、M6 以降の拡張が楽になる
- NPC の `id` をキーに会話データを引く構造にすると、NPC 追加がデータ追加だけで済む
- 前方 1 タイル判定を会話・イベントの共通基盤にできる
- `index.html` の script 読み込み順序が重要（例: `map.js` → `npc.js` → `interaction.js` → `collision.js`）
