# Mini RPG Dev — Agent 依頼テンプレート集

Cursor Agent へ依頼するときにコピー＆ペーストして使うテンプレートです。  
`[ ]` 内を具体的な内容に置き換えてください。

関連: [AI_WORKFLOW.md](./AI_WORKFLOW.md)

---

## 1. 新機能追加

```
[機能名（例: M8 複数行会話）] を実装してください。

現在の状態：
- [実装済み機能を列挙（例: M0 主人公移動、M1 マップ表示、…）]
- [関連ファイル（例: dialogue.js に会話データあり）]

今回の仕様：
- [追加する機能の詳細]
- [ファイル構成の方針（新規 / 既存修正）]
- [今回やらないこと（任意）]

完了条件：
- [チェック項目 1]
- [チェック項目 2]
- [チェック項目 3]
- 最後に変更したファイルと理由を説明してください

既存機能を壊さないこと。
- [維持すべき機能（例: 移動、壁判定、カメラ、NPC、会話）]
```

## 2. バグ修正

```
[症状の概要（例: 会話ウィンドウが表示されない）] を修正してください。

症状：
- [実際に起きていること]

再現方法：
1. [手順 1]
2. [手順 2]
3. [手順 3]

期待する動作：
- [正しく動いたときの状態]

原因を調査して修正してください。

確認してほしいこと：
- [調査してほしいファイルや処理（任意）]

完了条件：
- [修正後に満たすべき条件]
- 最後に原因と修正内容を説明してください

既存機能を壊さないこと。
```

## 3. リファクタリング

```
[対象（例: script.js のゲームループ部分）] をリファクタリングしてください。

対象：
- [ファイル名・関数・モジュール]

目的：
- [整理の意図（例: 責務の分離、可読性向上、重複削除）]

動作は変更しないこと。
- [維持すべき挙動（例: 移動速度、当たり判定、描画順）]

コードを整理してください。
- [方針（例: 関数ごとに役割を分ける、命名を統一する）]

完了条件：
- 見た目・操作・挙動がリファクタリング前と同じ
- 最後に変更したファイルと理由を説明してください
```

## 4. ドキュメント作成

```
[ファイル名（例: CHANGELOG.md）] を新規作成してください。

作成するファイル：
- [ファイルパス]

目的：
- [このドキュメントの用途]

内容：
- [含める章・項目]
- [形式（例: Keep a Changelog 形式、日付ごとのログ）]
- [記載してほしい具体例（任意）]

コードは変更しないこと。
[Markdown のみ作成 / 既存コードは変更しない、と明記]

完了条件：
- [ファイルが作成されている]
- [指定した内容が含まれている]
```

---

## 使い方のヒント

| テンプレート | 向いている場面 |
|-------------|---------------|
| 新機能追加 | M0, M1 … のようなマイルストーン実装 |
| バグ修正 | 表示崩れ・入力不具合・例外エラー |
| リファクタリング | 動作を変えずにコード整理 |
| ドキュメント作成 | ROADMAP / DEVLOG / CHANGELOG など |

依頼後は [AI_WORKFLOW.md](./AI_WORKFLOW.md) に従い、動作確認 → ログ更新 → commit → push を行ってください。
